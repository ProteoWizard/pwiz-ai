# Tutorial Workflow Guide

End-to-end loop for editing a Skyline tutorial: HTML changes → tutorial
test → screenshot diff → accept/revert/fix. This guide is a **bridge** —
each step links to the canonical doc for details.

For a list of every tool involved, see the tables in
[`tutorial-doc-style-guide.md`](tutorial-doc-style-guide.md) and
[`screenshot-update-workflow.md`](screenshot-update-workflow.md).

## File locations

| Asset | Path |
|-------|------|
| Tutorial HTML | `pwiz_tools/Skyline/Documentation/Tutorials/{TutorialName}/{en\|ja\|zh-CHS}/` |
| Tutorial tests | `pwiz_tools/Skyline/TestTutorial/` |
| Shared CSS/JS | `pwiz_tools/Skyline/Documentation/Tutorials/shared/` |
| ImageComparer GUI | `pwiz_tools/Skyline/Executables/DevTools/ImageComparer/` |
| Diff images (working) | `ai/.tmp/{Tutorial}-{Locale}-s-{Number}-diff-{PixelCount}px.png` |

## The loop

### 1. Edit the tutorial HTML

- Style: [`tutorial-doc-style-guide.md`](tutorial-doc-style-guide.md) —
  standard template, action bullet conventions, menu pattern, locale notes.
- The `tutorial-documentation` skill auto-loads when working here.

### 2. Build Skyline

- See [`build-and-test-guide.md`](build-and-test-guide.md). The tutorial
  test executes the **last compiled binary**, so build before running the
  test — `TESTING.md` covers this rule under "Build Before Testing".

### 3. Run the tutorial test in auto-screenshot mode

- Test project: `TestTutorial.csproj` (base class
  `AbstractFunctionalTestEx`). See [`TESTING.md`](../TESTING.md) for
  project selection and the tutorial-test family.
- Auto-screenshot mode produces a fresh set of `s-XX.png` files into the
  tutorial's locale folder, replacing the on-disk baseline.

### 4. Diff the screenshots

Two paths, depending on what you want:

| Goal | Tool |
|------|------|
| Programmatic / batch review from Claude Code | **ImageComparer MCP server** — see [`mcp/image-comparer.md`](mcp/image-comparer.md). Tools: `list_changed_screenshots`, `generate_diff_image`, `generate_diff_report`, `revert_screenshot`. Always use forward-slash paths. |
| Interactive review with keyboard navigation | **ImageComparer GUI app** — see [`screenshot-update-workflow.md`](screenshot-update-workflow.md) (§ "ImageComparer Tool" for shortcuts). |

Diff images land in `ai/.tmp/` with the
`{Tutorial}-{Locale}-s-{Number}-diff-{PixelCount}px.png` pattern. The
`skyline-screenshots` skill auto-loads for this stage and recognizes the
short-form conventions (`s-06 has an issue`, `DIA s-10 accepted`,
`s-06, s-08 BUG-001`).

### 5. Decide per screenshot

- **Accept** — the change is correct; commit the new `s-XX.png`.
- **Revert** — restore the baseline via `revert_screenshot` (MCP) or `F12`
  (GUI). Use when the diff is noise or unintended.
- **Fix** — file/tag a bug (`BUG-XXX`); the screenshot stays out of the
  commit until the fix lands.

See [`screenshot-update-workflow.md`](screenshot-update-workflow.md)
§ "Decisions" for the full triage flow.

### 6. Cover shots

`cover.png` is a composite generated by `TakeCoverShot()` in the tutorial
test. Tests with overrides — DdaTutorialTest, DiaUmpireTutorialTest,
DiaSearchTutorialTest, AcquisitionComparisonTutorialTest,
ExistingExperimentsTutorialTest — must capture and return
`base.ProcessCoverShot(bmp)` on Win11 (it returns a new Bitmap, not the
original). See the
[`skyline-screenshots`](../claude/skills/skyline-screenshots/SKILL.md)
skill notes and `screenshot-update-workflow.md` § "Cover Shot System".

## Win11 machine setup

For consistent screenshots:

- Transparency effects: **OFF**
- Display scaling: **100%** (96 DPI)
- Mode: **Light**

Details in the [`skyline-screenshots`](../claude/skills/skyline-screenshots/SKILL.md)
skill.
